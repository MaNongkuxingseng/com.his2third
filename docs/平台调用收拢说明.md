# 平台调用收拢说明

## 当前状态

平台调用已经收敛为 `PlatformInvoker -> PlatformAdapter`。公共字典存在专用适配器时使用专用适配器，未配置时统一使用 `DefaultPlatformAdapter`，不再从 Target 获取平台声明。

## 设计约束

- `TargetRegistration` 是平台声明唯一来源，字段包括 `PlatformCode`、`PlatformClass`、`MethodName`、`PlatformData`。
- `Target` 只负责 payload、响应解析、成功判断和过滤，不再实现 `GetInvoke()`。
- `BasePushTemplate` 只保留 `Execute(ctx)` 模板入口，执行编排集中在 `PushExecutor`。
- `bizInfo` 统一归一化为 `data.id`、`data.no`、`ext`；历史字段仅在边界兼容。

## 默认适配器规则

- `PlatformData.NeedCode=1` 时调用 `(PlatformCode, Payload)`。
- `HIPMessageInfo` 且存在 `PlatformData.opCode` 或 `PlatformData.dataFormat` 时调用 `(PlatformCode, Payload, dataFormat, opCode)`。
- `PlatformClass` 包含 `PUB0005Soap` 时按四参 `HIPMessageInfo` 处理，避免字典缺失时退化成错误签名。
- 其他场景默认调用 `(Payload)`。
- 专用 adapter 可覆盖响应预处理；默认 adapter 不改写平台返回。
