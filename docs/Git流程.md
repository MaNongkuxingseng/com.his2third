# Git 流程

## 分支约定

- `main`：稳定基线，保存从 `PHABizPush.xml` 提取后的初始源码快照。
- `dev`：日常开发、文档补充和后续整理分支。

所有工程内变更先进入 `dev`，确认后再合并到 `main`。

## 提交规范

提交信息使用中文，建议格式：

```text
类型：简要说明
```

常用类型：

- `文档`：README、规范、导入说明等文档变更。
- `修复`：业务缺陷修复。
- `重构`：不改变行为的结构调整。
- `功能`：新增业务入口、Target、平台适配等能力。
- `配置`：Git、换行、工具配置等变更。

示例：

```text
文档：补充工程结构与开发规范
修复：调整门诊退药数量转换逻辑
配置：规范 cls 文件换行
```

## 提交流程

1. 确认当前分支为 `dev`。
2. 修改前查看 `git status`，避免混入无关文件。
3. 完成后检查差异，确认没有误改业务源码。
4. 提交时使用中文提交信息。
5. 合并回 `main` 前，应说明验证方式和未验证项。

## 本地安全目录提示

如果 Git 提示 `detected dubious ownership`，可对当前仓库添加安全目录配置。该配置只解决本机 Git 权限识别问题，不代表源码变更。
