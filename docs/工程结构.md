# 工程结构

## 总览

```text
HIS2SA/
├── .gitattributes
├── README.md
├── docs/
└── PHA/
    ├── COM/BaseBizPush/
    └── FACE/TPS/SA/HIS2SA/
```

`PHA.COM.BaseBizPush` 是通用推送框架，`PHA.FACE.TPS.SA.HIS2SA` 是面向 SA 系统的业务实现。两者应保持清晰边界：框架层不直接绑定具体业务表，业务层不重复实现通用编排能力。

## BaseBizPush

```text
PHA/COM/BaseBizPush/
├── App/          # 查询可推送目标、人工执行等应用服务
├── Business/     # 推送服务入口与业务模板基类
├── DTO/          # 前端或应用层传输对象
├── Engine/       # 上下文、注册、过滤、平台调用、结果、目标、模板
├── Manual/       # 人工重推请求、查询、执行和门面
└── Test/         # 目标测试工具
```

主要职责：

- `Engine.Context`：保存一次推送的入口、院区、科室、业务信息和目标限制。
- `Engine.Template`：负责注册匹配、过滤、同步或异步执行。
- `Engine.Target`：定义目标系统推送接口。
- `Engine.Platform`：封装平台调用方式和调用上下文。
- `Engine.Result`：组织推送结果和日志写入。

## HIS2SA

```text
PHA/FACE/TPS/SA/HIS2SA/
├── BusinessEntry/ # 业务入口：门诊、住院、草药发退药等
├── DTO/           # SA 请求数据对象
├── Data/          # HIS 数据查询和组装
├── Platforms/     # SA 平台适配器
└── Target/        # 具体推送目标
```

主要职责：

- `BusinessEntry`：声明入口编码、入口名称、注册目标和业务数据获取方式。
- `Data`：按业务 ID 查询 HIS 数据并组装为目标需要的动态对象。
- `DTO`：承载发药、退药等批量请求结构。
- `Platforms.Adapter`：适配 PUB0002、PUB0005 等平台调用差异。
- `Target`：构建 payload、调用平台、解析响应、判断成功与失败消息。

## 当前注意点

- 源码中保留了测试方法和 TODO 标记，它们属于后续整理对象，不在本次规范化中删除。
- `Storage Default` 来自原 IRIS 导出，修改 DTO 属性时需同步确认存储定义。
- 当前仓库没有 IRIS 在线编译校验配置，代码导入后应在目标命名空间完成编译确认。
