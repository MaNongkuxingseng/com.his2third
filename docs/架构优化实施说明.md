# 架构优化实施说明

## 本次落地范围

- 新增入口解析、上下文工厂、目标工厂、注册读取、执行器、过滤管线、平台调用器等框架层服务。
- 保持 `PushService.Push`、`PushManualFacade.QueryTargets`、`PushManualFacade.ExecuteByDynamic` 等外部入口兼容。
- 修复 `deptId/deptCode` 混用、`PlatformInvoke` 伪动态调用、`PushRegistry` 旧字段读取、`CheckDataFilte` 拼写兼容等问题。
- 新增日志接口和平台适配器解析接口，默认实现仍复用现有日志表和公共字典。

## 新增内核分层

- `Engine.Resolve`：处理 entryCode 到 Entry 类的解析与缓存。
- `Engine.Factory`：处理 Context 与 Entry/Target 实例化。
- `Engine.Execute`：处理推送执行编排、过滤、平台调用、异步调度和结果构造。
- `Engine.Result`：保留结果对象，同时新增日志接口，避免 `PushSummary` 直接依赖日志基础设施。
- `Engine.Platform.Adapter`：平台适配器解析接口与字典实现。
- `Engine.Target`：保留 `ITarget` 兼容门面，新增小接口和 `BaseTarget` 供后续 Target 渐进迁移。

## 后续迁移建议

- 新 Target 优先继承 `BaseTarget`，只覆写必要方法。
- 新 Data 方法优先返回 `PHA.FACE.TPS.SA.HIS2SA.Data.DataResult.Ok/Fail` 结构。
- 消耗、退药类 DTO 输出可逐步迁移到 `BatchJsonMapper`，减少 Header + Items 重复拼装。
- 旧测试方法暂不删除，后续集中迁移到 `PHA.COM.BaseBizPush.Test`。
